# AI Project — 에이전트 팀 가이드

## 팀 구성

| 명령어 | 역할 | 언제 사용 |
|--------|------|-----------|
| `/pm` | 기획자 (Product Manager) | 만들 기능을 설명할 때 |
| `/dev` | 개발자 (Developer) | 실제 코드를 만들 때 |
| `/review` | 코드 리뷰어 | 완성된 코드를 점검할 때 |
| `/qa` | QA 엔지니어 | 기능이 제대로 되는지 확인할 때 |
| `/full-cycle` | 전체 워크플로우 | 기능 하나를 처음부터 끝까지 만들 때 |

## 권장 작업 순서

```
1. /pm       → 기능 요구사항 정의
2. /dev      → 코드 구현
3. /review   → 코드 품질 점검
4. /qa       → 기능 검증
```

또는 한 번에: `/full-cycle` 실행

## 이 프로젝트에 대해

- 대상: 비개발자가 Claude Code와 함께 만드는 웹 서비스
- 언어: 한국어로 소통, 코드는 영어
- 원칙: 복잡한 설명 없이 명확하고 실행 가능한 결과물 중심
